继承的几种方式

在 JavaScript 中，实现继承的方式主要有以下几种，每种方式有不同的优缺点：

1.原型链继承（Prototype Inheritance）

通过将子类的原型指向父类的实例，来实现继承。

function Animal(name) {
  this.name = name;
}

Animal.prototype.sayHello = function() {
  console.log(`${this.name} says hello!`);
};

function Dog(name) {
  Animal.call(this, name);  // 继承父类的属性
}

Dog.prototype = new Animal();  // 继承父类的方法
Dog.prototype.constructor = Dog;  // 修正constructor指向

const dog = new Dog('Buddy');
dog.sayHello();  // "Buddy says hello!"

优点：

  • 简单易懂，适用于轻量级的继承。
  • 子类能够继承父类实例的属性和方法。

缺点：

  • 子类实例会共享父类实例的属性（即原型链上的共享对象），这可能导致不希望发生的副作用。
  • 子类实例无法独立地修改父类的属性，修改会影响到其他实例。
  • 无法继承父类的私有属性和方法。

适用场景：

  • 适用于简单的对象继承场景，比如原型链上的继承不需要对静态方法或复杂的构造函数逻辑进行继承的情况。

2.构造函数继承（Constructor Inheritance）

通过在子类构造函数中调用父类构造函数，实现继承。

function Animal(name) {
  this.name = name;
}

Animal.prototype.sayHello = function() {
  console.log(`${this.name} says hello!`);
};

function Dog(name) {
  Animal.call(this, name);  // 继承父类的属性
}

const dog = new Dog('Buddy');
dog.sayHello();  // "Buddy says hello!"

优点：

  • 可以继承父类的实例属性，避免了原型链继承中共享实例属性的副作用。

缺点：

  • 只能继承父类的实例属性，无法继承父类的方法。
  • 每次创建子类实例时，都会调用父类的构造函数，可能造成性能浪费（如果父类构造函数较重）。

适用场景：

  • 适合需要继承父类实例属性，但不需要继承父类方法的场景。

3.组合继承（Combination Inheritance）

结合了原型链继承和构造函数继承，既能继承父类的实例属性，又能继承父类的方法。

function Animal(name) {
  this.name = name;
}

Animal.prototype.sayHello = function() {
  console.log(`${this.name} says hello!`);
};

function Dog(name, breed) {
  Animal.call(this, name);  // 继承父类的属性
  this.breed = breed;
}

Dog.prototype = Object.create(Animal.prototype);  // 继承父类的方法
Dog.prototype.constructor = Dog;  // 修正constructor指向

const dog = new Dog('Buddy', 'Golden Retriever');
dog.sayHello();  // "Buddy says hello!"

优点：

  • 既可以继承父类的实例属性，又能继承父类的方法，避免了原型链继承的副作用。
  • 解决了原型链继承中子类实例共享父类实例属性的问题。

缺点：

  • 需要调用两次父类构造函数（一次在原型链继承中，一次在子类构造函数中），可能会造成性能问题，尤其在构造函数较重时。

适用场景：

  • 适合较为复杂的继承结构，子类需要同时继承父类的实例属性和方法，并且不介意多次调用父类构造函数的性能问题。

4.ES6 class 语法继承

ES6 引入了 class 语法，可以通过 extends 关键字实现继承。

class Animal {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(`${this.name} says hello!`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name);  // 调用父类构造函数
    this.breed = breed;
  }
}

const dog = new Dog('Buddy', 'Golden Retriever');
dog.sayHello();  // "Buddy says hello!"

优点：

  • 语法简洁、易读。
  • super() 方法确保了父类构造函数的正确调用，并且清晰地表达了继承关系。
  • 可以更容易地处理父类和子类之间的构造函数、静态方法、继承等细节。

缺点：

  • 内部实现仍然依赖原型链和构造函数继承，但语法层面对开发者更加友好。
  • 可能需要兼容老旧浏览器（不支持 ES6）。

适用场景：

  • 适合现代 JavaScript 开发，尤其是在需要清晰表示继承关系和更好组织代码的场景。

5.ES6 Object.create() 继承

通过 Object.create() 方法直接基于父类的原型创建子类实例。

const Animal = {
  sayHello() {
    console.log(`${this.name} says hello!`);
  }
};

const dog = Object.create(Animal);
dog.name = 'Buddy';
dog.sayHello();  // "Buddy says hello!"

优点：

  • 非常灵活，可以创建没有构造函数的“类”。
  • 直接操作原型链，避免了不必要的构造函数调用。

缺点：

  • 没有构造函数，无法直接添加实例属性，只能通过原型链继承。
  • 需要手动管理实例属性，可能导致代码不够清晰。

适用场景：

  • 用于一些特殊需求的继承场景，适合需要灵活继承且不需要复杂构造函数的情况。

6.寄生继承（Parasitic Inheritance）

寄生继承是通过创建一个新对象并将其返回，从而将父类的属性和方法“复制”到子类对象中。这种方式类似于组合继承，但它的目标是构造一个新对象来作为子类原型。

function Animal(name) {
  this.name = name;
}

Animal.prototype.sayHello = function() {
  console.log(`${this.name} says hello!`);
};

function Dog(name, breed) {
  const dog = new Animal(name);  // 创建父类实例
  dog.breed = breed;  // 添加新属性
  return dog;  // 返回新对象
}

const dog = new Dog('Buddy', 'Golden Retriever');
dog.sayHello();  // "Buddy says hello!"
console.log(dog.breed);  // "Golden Retriever"

优点：

  • 继承了父类的实例属性和方法。
  • 代码相对简洁。

缺点：

  • 无法继承父类的原型方法（比如 sayHello），这种方式更多的是创建一个新对象，而不是让子类继承父类的完整结构。
  • 对象的原型链不再是一个真正的继承链。

适用场景：

  • 在需要继承父类的属性和方法时，但不需要原型链结构时，可以使用这种方式。

7.ES6 Object.setPrototypeOf() 继承

通过 Object.setPrototypeOf() 方法动态设置对象的原型，从而实现继承。

const Animal = {
  sayHello() {
    console.log(`${this.name} says hello!`);
  }
};

const dog = {
  name: 'Buddy'
};

Object.setPrototypeOf(dog, Animal);  // 设置原型链

dog.sayHello();  // "Buddy says hello!"

优点：

  • 动态修改原型链，能够更灵活地控制继承关系。
  • 简洁明了，使用原生方法。

缺点：

  • 修改原型的性能较差，尤其是频繁修改时，可能会导致性能下降。
  • 对象的原型链可能会变得不清晰，影响代码的可维护性。

适用场景：

  • 在需要动态调整对象原型的特殊场景中可以使用，但一般不推荐用于频繁修改原型的场景。

8.Mixins（混入）

Mixins 是一种组合多个对象的属性和方法的方式。虽然它不是传统意义上的继承，但它提供了一种代码复用的方式，可以在多个对象之间共享功能。

const Animal = {
  sayHello() {
    console.log(`${this.name} says hello!`);
  }
};

const Walker = {
  walk() {
    console.log(`${this.name} is walking!`);
  }
};

function createDog(name) {
  const dog = Object.create(Animal);
  dog.name = name;
  Object.assign(dog, Walker);  // 将 Walker 的方法混入 dog 对象
  return dog;
}

const dog = createDog('Buddy');
dog.sayHello();  // "Buddy says hello!"
dog.walk();  // "Buddy is walking!"

优点：

  • 可以混入多个对象的方法和属性，达到代码复用的目的。
  • 不需要构建复杂的继承体系。

缺点：

  • 如果混入的对象之间存在命名冲突，可能会引发问题。
  • 不是真正的继承，只是功能的组合，可能导致代码的组织结构不够清晰。

适用场景：

  • 适合功能复用而不需要层级关系的场景，尤其是在多个对象之间共享某些行为时，如 Vue 的 mixins。

9.委托继承（Delegation Inheritance）

委托继承是一种通过委托机制将请求转发给其他对象的方式。在这种模式下，子类并不直接继承父类的属性和方法，而是通过某种形式将请求委托给父类来实现行为复用。

function Animal(name) {
  this.name = name;
}

Animal.prototype.sayHello = function() {
  console.log(`${this.name} says hello!`);
};

function Dog(name, breed) {
  const animal = new Animal(name);  // 委托给 Animal 创建实例
  this.sayHello = animal.sayHello;  // 委托方法
  this.breed = breed;
}

const dog = new Dog('Buddy', 'Golden Retriever');
dog.sayHello();  // "Buddy says hello!"

优点：

  • 子类只需要委托父类的行为即可，不需要直接继承。
  • 可以节省内存，因为方法是通过委托的方式共享的。

缺点：

  • 不是真正的继承，不能使用传统继承中的原型链机制。
  • 不够灵活，可能在一些复杂的继承需求中不适用。

适用场景：

  • 用于特定场景下的委托模式，通常会与组合模式结合使用。
