CSS变量的性能

sleepkele2025/2/8大约 8 分钟vue红小圆

CSS变量的性能

CSS变量已经存在一段时间了。尽管它们仍然缺乏一些浏览器的支持（主要是IE和Edge），但它们提供了一种全新的方式来组织和交互CSS。为了更深入地了解CSS变量对性能的影响，我们必须理解作用域的基本原理，因为它同样适用于CSS变量。

太长不看

  • 注意样式重新计算，因为 CSS 变量是可继承的——更改父元素上的一个变量可能会影响多个子元素。
  • 建议元素使用单个类，以便浏览器更轻松地进行样式计算。
  • calc()处理变量时性能良好，但对某些单位的浏览器支持仍然存在问题，例如deg or ms
  • 在 Javascript 中，建议使用setProperty 内联样式而不是内联样式来设置 CSS 变量。

全球范围与本地范围

作用域的概念存在于大多数编程语言中，它本质上定义了变量是在整个文档中生效，还是仅在文档的某个子部分（例如 CSS 类）中生效。对于 CSS 变量，这意味着我们可以像这样在全局作用域中定义变量：

:root {
	--main-color: tomato;
}

如果我们希望变量仅对特定组件可见，我们可以将其定义在该组件的局部作用域中。

.title {
	--title-color: aqua;
	color: var(--title-color);
}

CSS 变量的优点在于它们的动态性，因此可以在不同的作用域和运行时进行更改。所以，我们不必像之前那样定义一个新变量--title-color，只需更改现有的全局变量即可，而且更改只会影响这个特定的组件。

.title {
  --main-color: aqua;
	color: var(--main-color);
}

范围和具体性

借助 SASS 等工具，嵌套 CSS 变得非常流行，但这往往会导致出现像这样的过于具体的选择器：

.card .heading .title { ... }

为了解决这个问题，可以使用以类为中心的方法，例如边界元已经引入了这种方法。它的目的是通过为每个元素使用单个类来降低类型特异性。

.card { ... }
.card__heading { ... }
.card__title { ... }

继承和样式重新计算

为了了解作用域对性能的影响，我们首先需要了解浏览器如何处理样式计算。

例如，为了知道某个元素是其类型的最后一个元素，浏览器必须首先了解所有其他元素，以及它后面是否有倒数第n个子元素，这可能比仅仅因为类匹配而将选择器与元素匹配要昂贵得多。刘易斯 2017）

使用像 BEM 那样的单一类，可以大大简化浏览器的样式计算，因为它不需要将元素与其所有同级元素进行比较来确定要应用的样式。

作用域和 CSS 变量

由于 CSS 变量是可继承的，因此更改它们会导致所有子元素的样式重新计算。这不仅适用于 CSS 变量，也适用于其他可继承的属性，例如颜色或字体。

通常情况下，默认继承的 CSS 属性在大多数情况下会导致样式的大量重新计算。……避免这种情况的一个好做法是在最具体的级别（或最深的级别）对 CSS 变量进行动画处理，以防止影响到大量的子元素。盖贝尔 2017）

因此，如果我们要动态更改 CSS 变量，应该在局部作用域而不是全局作用域中进行更改，以避免大规模的样式重新计算。

样式重新计算示例

为了衡量在拥有 25000 个子元素的父元素上设置 CSS 变量与仅在子元素上设置变量之间的差异，我们创建了一个 Codepen，其中包含以下 HTML 和 CSS 代码。

<div class="container">
  <span class="el"></span>
... 25000 more elements
</div>
.el {
	background: var(--bg, orange);
}

首先，通过 JavaScript--bg在父元素上设置了该变量.container，这导致了长达 76 毫秒的执行时间。然后，在第一个子元素 .el 上设置了相同的变量，这次只持续了大约 1.9 毫秒。因此，父元素使用此变量的子元素越多，在该元素上设置 CSS 变量的开销就越大。

将 Calc() 与 CSS 变量结合使用

当 CSS 变量与函数结合使用时，它们会变得更加强大calc()。尤其是在变换方面，使用一个无单位变量定义多个属性，然后根据这些属性的单位设置它们的值，这种方式更优，也更容易调整这些过渡效果。威尔逊 2017到目前为止，width 和 calc()height 主要用于计算响应式元素的宽度和高度。例如，如果想要将一个 div 的宽度设置100px为小于某个值100%，可以这样做：

.container {
  width: calc(100% - 100px);
}

结合 CSS 变量，calc()可以实现更多功能。给一个没有单位的 CSS 变量添加单位可以这样做：

:root {
	--duration: 2000;
}
div {
  animation-duration: calc(var(--duration) * 1ms);
}

使用 calc() 函数的性能

当与无单位 CSS 变量结合使用时calc()，性能可能会出现问题。要了解更多关于性能问题的信息，请参阅……jsPerf 测试用例创建此程序是为了测试设置属性时使用和不使用参数的几种不同方法calc()。以下列出了六种测试方法：
变量 + calc + px：使用 calc() 将无单位的 CSS 变量乘以 1px。
变量 + px：设置一个包含像素值的 CSS 变量
px：设置一个硬编码的像素值
变量 + 计算 + 百分比：使用 calc() 将无单位的 CSS 变量乘以 1%。
变量 + 百分比：设置一个包含百分比值的 CSS 变量
百分比：设置了一个硬编码的百分比值
如下所示，直接设置硬编码值最快（方法 3 和 6），其次是设置包含带单位值的 CSS 变量（方法 2 和 5）。借助某种方法设置无单位 CSS 变量calc()（方法 1 和 4）最慢，但与前一种方法相比慢不了多少。

浏览器支持

部分浏览器在支持 x 时仍然存在问题calc()。好的一方面是，所有支持 CSS 变量的浏览器也都支持 calc()xpx, vmin, rem和其他线性距离单位。另一方面，Firefox 和 Edge 在使用其他单位类型（例如 %, degx 或 y）时存在问题ms。这些问题阻碍了calc()许多开发者使用 CSS 变量进行开发，但渐进增强技术允许在这些受影响的浏览器中使用回退方案。威尔逊 2017）。

使用 JS 设置 CSS 变量

David Khourshid 创建了一个jsPerf 测试用例在文中，他比较了三种使用 JavaScript 设置样式的不同方法。通过调整测试，我们比较了以下几种情况。jsPerf 测试：

首先，设置内联样式：

element.style = ’color: green’;

其次，样式是使用 setProperty 语法设置的：

element.style.setProperty(’color’, ’green’);

第三种方法也使用了 setProperty，但用于设置 CSS 自定义属性：

element.style.setProperty(’--color’, ’green’);

最后一个测试用例将 CSS 变量设置为内联样式：

element.style = "--color: green";

测试用例多次应用于 100 个 HTML 元素。Safari 在设置内联样式方面比element.style.setProperty直接设置 CSS 变量快得多。而 Chrome 中，通过内联样式或直接设置 CSS 变量的速度差异不大setProperty。有趣的是，Firefox 在设置内联样式方面明显落后。因此，在setProperty所有浏览器中，直接设置样式都是更优的选择。在所有三个浏览器中，直接设置样式都比设置变量更快。

结论

在使用 CSS 变量时，我们必须始终注意变量的定义范围，如果更改它，将会影响许多子元素，从而导致大量的样式重新计算。

结合 CSS 变量使用calc()可以有效提高灵活性，并减少需要定义的变量数量。calc()在不同浏览器中测试结合 CSS 变量使用后，并未发现明显的性能问题。然而，部分浏览器对某些单位（例如度或毫秒）的支持仍然有限，因此我们需要注意这一点。

如果我们比较通过内联样式和使用其他方法在 JavaScript 中设置变量的性能表现，setProperty会发现不同浏览器之间存在一些显著差异。虽然在 Safari 中通过内联样式设置属性非常快，但在 Firefox 中却很慢，因此使用其他方法setProperty设置变量是更优的选择。
