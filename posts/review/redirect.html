实现重定向到多种方式与第三方登录、支付宝支付、微信支付大致流程介绍

一、前端框架实现

  1. Vue 路由重定向
src/router/index.js
const routes = [
  {
    path: '/old',
    redirect: '/new' // 直接重定向
  },
  {
    path: '/old2',
    redirect: { name: 'NewPage' } // 命名路由重定向
  }
]
  1. React 路由重定向
src/App.jsx
// 类组件方式
<Redirect from="/old" to="/new" />

// 函数组件内
import { useHistory } from 'react-router-dom';
function MyComponent() {
  const history = useHistory();
  history.push('/new-page');
}

二、原生 JavaScript

// 普通重定向
window.location.href = 'https://newdomain.com';

// 替换当前历史记录
window.location.replace('https://newdomain.com');

// 定时重定向
setTimeout(() => {
  window.location = '/new-page';
}, 3000);

三、后端实现

  1. Node.js (Express)
server.js
app.get('/old', (req, res) => {
  res.redirect(301, '/new'); // 301永久重定向
});

// 带参数重定向
app.get('/search', (req, res) => {
  res.redirect(`/result?q=${req.query.keyword}`);
});
  1. Java Spring Boot

Controller.java

    RedirectView redirectView = new RedirectView();
    redirectView.setUrl("/new");
    return redirectView;
}
  1. Python Django
from django.shortcuts import redirect

def old_view(request):
    return redirect('/new', permanent=True)

axios.interceptors.response.use(response => {
  return response;
}, error => {
  if (error.response.status === 302) {
    window.location = error.response.headers.location;
  }
  return Promise.reject(error);
});
  1. 微信小程序重定向

建议：

  1. 前端路由：适用于SPA应用内部导航
  2. 服务端重定向：适合SEO优化、域名迁移等场景
  3. 301/302状态码：
    • 301：永久重定向（SEO权重转移）
    • 302：临时重定向（保留原页面权重）
  4. 服务器配置：适合全站级别的重定向规则
  5. 需要根据具体场景（SEO需求、登录验证、页面迁移等）选择合适的重定向方式。

第三方登录

通用实现方案及主流平台的技术

一、通用实现原理

sequenceDiagram
    User->>Client: 点击第三方登录
    Client->>OAuth Server: 跳转授权页面
    OAuth Server-->>User: 显示授权页面
    User->>OAuth Server: 授权登录
    OAuth Server->>Client: 返回授权码
    Client->>Server: 发送授权码
    Server->>OAuth Server: 用授权码换令牌
    OAuth Server->>Server: 返回访问令牌
    Server->>Client: 创建本地会话

二、具体实现步骤（以QQ登录为例）

  1. 前端处理
Login.vue
<template>
  <div>
    <!-- 直接使用平台提供的登录按钮 -->
    <a href="https://graph.qq.com/oauth2.0/authorize?response_type=code&client_id=YOUR_APP_ID&redirect_uri=YOUR_CALLBACK_URL">
      <img src="qq_login_button.png">
    </a>
  </div>
</template>
  1. 后端处理（Node.js示例）
oauth.js
router.get('/qq/callback', async (req, res) => {
  // 1. 获取授权码
  const { code } = req.query;
  
  // 2. 换取访问令牌
  const tokenRes = await axios.get(
    `https://graph.qq.com/oauth2.0/token?grant_type=authorization_code&client_id=${APP_ID}&client_secret=${APP_KEY}&code=${code}&redirect_uri=${CALLBACK_URL}`
  );

  // 3. 获取用户OpenID
  const openidRes = await axios.get(
    `https://graph.qq.com/oauth2.0/me?access_token=${tokenRes.data.access_token}`
  );

  // 4. 获取用户信息
  const userInfo = await axios.get(
    `https://graph.qq.com/user/get_user_info?access_token=${tokenRes.data.access_token}&oauth_consumer_key=${APP_ID}&openid=${openidRes.data.openid}`
  );

  // 5. 创建/更新本地用户
  const user = await User.findOrCreate({ 
    where: { qq_openid: openidRes.data.openid },
    defaults: { ...userInfo.data }
  });

  // 6. 生成本地会话
  const token = jwt.sign({ userId: user.id }, SECRET_KEY);
  res.json({ token });
});

async function socialLogin(platform, openid) {
  // 查找现有关联
  const link = await SocialLink.findOne({ where: { platform, openid } });
  
  if (link) {
    return User.findById(link.userId);
  }
  
  // 新用户创建逻辑
  if (req.user) {
    // 已登录用户：绑定新平台

不同平台的核心差异：

  • 授权流程：微博使用OAuth2.0简化模式，GitHub使用标准授权码模式
  • 用户标识：QQ使用OpenID，GitHub使用数字ID，微信使用UnionID
  • HTTPS要求：微信、QQ等国内平台强制要求HTTPS回调地址

在Vue项目中实现支付功能通用方式及配合

一、通用支付流程

sequenceDiagram
    Vue->>+Backend: 1. 创建订单(商品信息)
    Backend->>+Payment: 2. 预支付请求
    Payment-->>-Backend: 返回支付参数
    Backend->>-Vue: 3. 返回支付参数
    Vue->>+PaymentSDK: 4. 调起支付界面
    PaymentSDK->>-Vue: 5. 支付结果(前端)
    Payment->>+Backend: 6. 异步通知(后端)

二、支付宝实现方案

  1. 前端代码
Payment.vue

<template>
  <div @click="handleAlipay">支付宝支付</div>
</template>

<script>
export default {
  methods: {
    async handleAlipay() {
      try {
        // 获取后端返回的支付参数
        const { data } = await this.$api.createAlipayOrder({
          amount: '100.00',
          subject: '证书服务'
        });

        // 动态加载支付宝JS文件
        const script = document.createElement('script');
        script.src = 'https://gw.alipayobjects.com/as/g/h5-lib/alipayjsapi/3.1.1/alipayjsapi.min.js';
        script.onload = () => {
          window.AlipayJSBridge.call('tradePay', {
            tradeNO: data.tradeNo // 后端返回的交易号
          }, (result) => {
            if (result.resultCode === '9000') {
              this.checkPaymentStatus(data.orderId);
            }
          });
        };
        document.body.appendChild(script);
        
      } catch (error) {
        this.$message.error('支付发起失败');
      }
    },
    async checkPaymentStatus(orderId) {
      // 轮询查询支付结果
      const res = await this.$api.checkOrderStatus(orderId);
      if (res.data.status === 'success') {
        this.$router.push('/pay-success');
      }
    }
  }
}
</script>

三、微信支付实现方案

Payment.vue

<template>
  <div ref="wxPay"></div>
</template>

<script>
export default {
  methods: {
    async initWechatPay() {
      // 获取微信配置参数
      const { data } = await this.$api.getWxConfigParams();
      
      // 初始化微信JS-SDK
      window.wx.config({
        debug: false,
        appId: data.appId,
        timestamp: data.timestamp,
        nonceStr: data.nonceStr,
        signature: data.signature,
        jsApiList: ['chooseWXPay']
      });

      window.wx.ready(() => {
        this.$refs.wxPay.addEventListener('click', this.executeWxPay);
      });
    },
    async executeWxPay() {
      const { data } = await this.$api.createWxOrder({
        total_fee: 100,
        body: '证书服务'
      });
      
      window.wx.chooseWXPay({
        timestamp: data.timeStamp,
        nonceStr: data.nonceStr,
        package: data.package,
        signType: 'MD5',
        paySign: data.paySign,
        success: () => {
          this.checkPaymentStatus(data.orderId);
        }
      });
    }
  },
  mounted() {
    this.initWechatPay();
  }
}
</script>

  1. 安全处理示例(Node.js)
payment.js

// 支付宝签名生成
const createAlipayParams = (order) => {
  const params = new Map([
    ['app_id', ALIPAY_APPID],
    ['method', 'alipay.trade.app.pay'],
    ['charset', 'utf-8'],
    ['sign_type', 'RSA2'],
    ['timestamp', moment().format('YYYY-MM-DD HH:mm:ss')],
    ['version', '1.0'],
    ['biz_content', JSON.stringify({
      total_amount: order.amount,
      subject: order.subject,
      out_trade_no: order.id
    })]
  ]);

  const sign = crypto.createSign('RSA-SHA256');
  sign.update([...params].sort().join('&'));
  return {
    ...Object.fromEntries(params),
    sign: sign.sign(ALIPAY_PRIVATE_KEY, 'base64')
  };
};

五、注意事项

  1. 安全规范：

    • 签名验证必须在后端完成
    • 支付金额使用字符串传递避免精度问题
    • 异步通知需验证支付平台身份

  2. 调试技巧：

  }
});
