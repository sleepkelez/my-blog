性能优化篇

sleepkele2026/4/15大约 2 分钟

性能优化篇

1. Avoid large layout shifts 避免大幅度的布局偏移

核心优化指标

  • 累积布局偏移（CLS） ‌：衡量页面加载过程中内容位置变化的程度，阈值为0.1（良好）、0.25（需改进）、0.25以上（不良）。
  • 最大内容绘制（LCP） ‌：确保主内容快速渲染，建议控制在2.5秒内。

优化方法

  • 优化图片加载
    • 避免低质量大图，使用 WebP格式 压缩图片
    • 图片懒加载（lazy-load）非首屏内容
  • 减少渲染阻塞资源
    • 将非关键CSS/JS代码延迟加载，或内嵌到HTML中
    • 预连接（preconnect）外部资源域名以提升加载速度
  • ‌代码与结构优化
    • 精简DOM结构，减少复杂动画和动态效果
    • 使用 CSS动画 替代部分 JavaScript 动画

2. Reduce unused CSS 减少不必要的CSS

  • 合并CSS文件：将多个CSS文件合并为一个文件可以减少HTTP请求的数量，从而加快页面加载速度
  • 使用CSS压缩工具：使用工具如CSSNano或CleanCSS可以移除多余的空白、注释和不必要的字符，从而减小CSS文件的大小
  • 移除未使用的选择器和规则：通过工具如PurgeCSS或Uncss可以帮助识别并移除未使用的CSS代码
  • 使用CSS精灵（CSS Sprites）：将多个图标合并到一个图像中，并通过CSS控制显示特定部分，可以减少HTTP请求的数量
  • 利用CSS预处理器：如Sass或Less，它们提供了变量、混合（Mixins）、函数等高级功能，可以帮助组织和管理CSS代码，但最终编译成较小的CSS文件。
  • 媒体查询优化：确保媒体查询（Media Queries）仅应用于需要它们的地方，避免在不支持媒体查询的旧浏览器上加载不必要的样式。
  • 使用内联关键CSS：对于关键渲染路径中的CSS，可以使用内联样式（inline styles）直接在HTML元素中定义，以减少关键渲染路径的长度。
  • 利用浏览器缓存：通过设置适当的缓存策略，可以让浏览器缓存已下载的CSS文件，减少重复下载。
  • 避免使用@import：虽然在一些情况下@import很有用，但它会增加额外的HTTP请求，特别是在移动网络环境中。尽可能使用链接到外部文件的<link>标签来代替@import。
  • 使用字体图标而非图片：使用字体图标可以减少HTTP请求的数量，并且可以像文本一样控制字体大小和颜色。

例：
使用PurgeCSS移除未使用的CSS

npm install purgecss --save-dev
const PurgeCSSPlugin = require('purgecss-webpack-plugin');
 
module.exports = {
  plugins: [
    new PurgeCSSPlugin({
      paths: glob.sync([
        path.join(__dirname, '*.html'), // 你的HTML文件路径
        path.join(__dirname, '*.js'), // 你的JavaScript文件路径（如果有内联样式）
      ]),
    }),
  ],
};
